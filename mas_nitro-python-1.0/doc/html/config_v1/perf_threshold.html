perf_threshold

Configuration for PerfThreshold resource.

Read/write properties

sms_profile<String>

SMS Profile.

clear_message<String>

Clear Message.

threshold_type<String>

Threshold Type.

id<String>

Id is system generated key for all the perf thresholds configuration .
Maximum length = 256

report_name<String>

Report Name for which the threshold is configured.
Minimum length = 1
Maximum length = 100

severity<String>

Severity.

mail_profile<String>

Mail Profile.

tenant_name<String>

Tenant Name.

is_enabled<Boolean>

enabled or disabled.

device_family<String>

Device Family of which this threshold belongs to.

instances<String>

counter instances.
Minimum length = 1
Maximum length = 100

threshold_value<Double>

Threshold Value.

name<String>

name.

duration<String>

duration of metric to be checked against threshold.

counter_name<String>

Counter Name for which the threshold is configured.
Minimum length = 1
Maximum length = 100

devices<String>

On which devices the threshold is enabled.

message<String>

Message.

clear_value<Double>

Clear Value.

devices_array<String[]>

Devices list.

instances_array<String[]>

Instances list.

Read only properties

clear_event<Boolean>

set to true if the event needs to be cleared.

The following parameters can be used in the nitro request :

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE
