ns_clag_interface

Configuration for CLAG Interface resource.

Read/write properties

channel_id<String>

Channel ID if this interface represents a channel (CLA/1, CLA/2 ...).
Minimum length = 5
Maximum length = 5

channel_tag_all_vlans<Boolean>

If true then all the member interfaces of this channel are tagged. Possible values: true and false.

channel_ha_monitoring<Boolean>

HA-monitoring control for the channel. Possible values: true and false.

lacp_channel_time<String>

LACP time. Possible values: long and short.
Minimum length = 4
Maximum length = 10

channel_interface_list<ns_clag_node[]>

Array of CLAG nodes that are part of this channel (10/1, 10/4).

clip<String>

Cluster IPAddress.
Minimum length = 1
Maximum length = 64

channel_alias<String>

Alias name for this channel.
Minimum length = 4
Maximum length = 31

has_foreign_interfaces<Boolean>

Indicates whether CLAG has interfaces from remote SDX. Possible values: true and false.

id<String>

Id is system generated key for CLAG.

mtu<Integer>

MTU value, should be between 1500-9216.
Minimum value = 1500
Maximum value =

sync_operation<Boolean>

sync operation.

Read only properties

lacp_mode<String>

LACP mode can be either ACTIVE, PASSIVE, DISABLED. Applicable for channel.

lacp_channel_mac_address<String>

Mac address of the bond on cluster.

channel_type<String>

Channel type to represent the CLAG type.

act_id<String>

Activity Id.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE
